# Non Functional Requirements

Помимо функциональных возможностей система должна соответствовать ряду нефункциональных требований.

## 1. Масштабируемость

- Система должна поддерживать увеличение количества домов и устройств.
- Должна быть возможность масштабировать отдельные сервисы независимо друг от друга.

## 2. Производительность

- Запрос текущего состояния устройства должен выполняться быстро (в разумных пределах для веб-приложения).
- Команды управления устройствами не должны приводить к заметным задержкам для пользователя.

## 3. Надёжность

- Отказ одного компонента системы не должен полностью останавливать работу всей экосистемы.
- Система должна корректно обрабатывать ошибки взаимодействия с устройствами.

## 4. Расширяемость

- Архитектура должна позволять добавлять новые типы устройств и модулей без серьёзной переработки системы.
- Должна быть возможность интеграции устройств партнёров.

## 5. Безопасность

- Доступ к системе должен быть защищён аутентификацией.
- Пользователь должен иметь доступ только к своим домам и устройствам.
